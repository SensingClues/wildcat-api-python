WildcatApi

Info

This document contains all the functions that can be called by the api.

  • Blue box is the class

  • grey box are the function you can call on the class

The api is a class that has to be initalized. Initializing this class is done with the user_name and password. Afterwards, you can use this class to call the different functions. Moreover,
api = WildcatApi(username,password) # initialize
api.get_groups() # get group data

observation_extractor(groups, **kwargs)[source]

Extract observations

This function takes the **kwargs argument. This means that extra arguments can be added. Those arguments are used to call subfunctions. The allowed extra arguments are added in the parameters. The group argument is mandotory, the rest is optional

Parameters

  • groups (str) – for example [“focus-project-7136973”]

  • bounds – dict with coordinates {“north”: 90, “south”: 90, “west”: 90,”east”:90”}

  • operator – The operator that will be put in the query, for example [“intersects”]

  • date_from – start date for the query

  • date_to – end date of the query

  • type_analysis – for example [“Observation”, “track”]

  • query_text – for example entityId: ‘exampleE

  • page_nbr – Which page number to start (call will recalc page number when page_length is given)

  • page_length – How many pages should a request contain

  • end_time – For a timestamp for end_date, format = T00:00:00-00:00”

  • start_time – For a timestamp for start_date, format = T00:00:00-00:00”

Returns

Df containing the observations according to the filters

Return type

DataFrame
